Two ways to run it

Pick a mode during installation — switchable later in Settings. Either way, speech-to-text is bundled and the app starts and stops its helper servers for you.

Local

no account

Fully offline. Bundled Whisper speech-to-text plus the minicpm-v vision model (pulled at install, ~5.5 GB) so screenshots work too.

  • No API key, no login, no network needed
  • Everything runs on your machine
  • Great for private or air-gapped use

Claude

smarter

Routes answers through your local claude CLI for sharper, more capable replies and native screenshot vision.

  • One-time claude login + a Claude subscription
  • No Anthropic API credits burned
  • Speech-to-text still bundled and local

→ Switch modes anytime in Settings → Dashboard. The bundled servers are managed automatically — nothing to start by hand.

Everything it does

Built for live customer and engineering calls. All premium features are unlocked in this fork.

🎧 One-button listening

The headphones icon starts system-audio capture and mic VAD in one click. [ME] / [THEM] tags are added automatically as the conversation flows.

⌨️ Manual AI on Ctrl+Space

Ask the AI about the latest transcript only when you want to. Nothing auto-fires, so you stay in control of what's sent.

🖱️ Right-click message actions

On any message in the conversation: Explain · Translate · Answer. Each runs a focused prompt, independent of the meeting-coach system prompt.

📸 Screenshots with vision

Capture any screen and the image is read by the AI — the local minicpm-v model, or Claude's native vision. The Screenshot button is always enabled.

🖥️ Multi-monitor selection fix

Screenshot selection now works correctly across monitors with mixed DPI — the captured snapshot is drawn as the overlay, so no more white screens on a second display.

🔓 All features unlocked

No license walls, no premium banners, no nag popovers. Everything works out of the box.

🧹 Delete all conversations

One-click wipe in the Chats tab for fast resets between meetings — skips the per-conversation confirmation flow.

♻️ Self-healing capture

Stale “Capture already running” errors recover automatically after ~250 ms instead of blocking you mid-call.

📦 Self-contained

Helper servers and speech-to-text ship inside the installer and start/stop with the app. The auto-updater is removed so the fork's patches stick.

FAQ & troubleshooting

Do I need an API key or account?

Not for Local mode — speech-to-text and the vision model are bundled and run entirely on your machine. Claude mode needs a one-time claude login and a Claude subscription, but never spends Anthropic API credits.

Windows says the app is unsafe / SmartScreen blocks it.

The build is unsigned, so Windows SmartScreen shows a warning on first run. Click “More info” → “Run anyway”. The source is fully open on GitHub if you'd rather build it yourself.

Can I switch between Local and Claude later?

Yes — pick a mode during install, then change it anytime in Settings → Dashboard. The bundled helper servers are managed automatically.

Pluely doesn't react to the other side's audio.

Check that Windows has the correct speaker / system-audio device selected and that audio is actually playing. WASAPI loopback returns silence when nothing is playing.

The audio-recording hotkey doesn't work.

Another app probably holds that hotkey. Change the Audio recording shortcut in Settings → Shortcuts.

Is this the official Pluely?

No. This is a personal Windows-only fork of iamsrikanthnani/pluely with workflow patches for live calls. It is not affiliated with Pluely.com.
