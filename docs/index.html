What's different from upstream

13+ patches on top of stock v0.1.9. Headline changes:

Unified listening button

The headphones icon starts both system-audio capture and mic VAD in one click. [ME] / [THEM] tags get added automatically. AI never auto-fires.

Manual AI on Ctrl+Space

Press Ctrl+Space to ask Claude about the latest transcript. Wrapped with a permissive prompt so it never refuses off-topic questions.

Independent dictaphone

Stereo WAV of mic + system audio, fully separate from the STT pipeline. Output: ~/Documents/Pluely Recordings/. Middle-click to open the folder.

Right-click message actions

On any message in the conversation popover: Explain · Translate · Answer. Each runs a focused prompt that ignores the meeting-coach system prompt.

Screenshot with vision

The Screenshot button is always enabled. Images are auto-injected into the AI request. With the bundled Claude Code proxy, screenshots are read by the Claude Code Read tool.

Multi-monitor selection fix

Tauri's transparent overlay used to render opaque on secondary monitors with mixed DPI. Now the captured snapshot is drawn directly as the overlay background.

All features unlocked

Master license switch — hasActiveLicense forced true, premium banners and license modals removed, supportsImages clamped so the Screenshot icon never greys out.

Exit + drag handle on panel

Quit and window-move buttons live directly on the panel. The drag handle is always active, no Get License nag popover.

Default to conversation view

The System Audio popover opens in full conversation view, not single-reply mode.

Delete all conversations

One-click wipe button in the Chats tab — bypasses the per-conversation confirmation flow for fast resets between meetings.

Auto-recover stuck capture

Stale "Capture already running" errors self-heal after ~250 ms (Rust-side start_system_audio_capture takes and aborts the existing task instead of refusing).

Auto-updater off

Stock updater would overwrite the fork's patches. Updater plugin is removed from the bundle.

Install (pre-built binary)

If you only want to use the fork, not modify it:

  1. Download the latest pluely_*.msi or pluely_*-setup.exe from the Releases tab (or build from source, see below).
  2. Run the installer.
  3. Open Settings → Dashboard and configure an AI provider:
    • Direct Anthropic API — recommended for native vision (screenshots).
    • Local Ollama — text-only unless a vision model (llava / llama3.2-vision) is loaded.
    • The bundled pluely-proxy setup — local Claude Code CLI, supports screenshots via the Read tool.
  4. Configure an STT provider — local Whisper via the proxy is the fastest path.

Build from source

Prerequisites

  • Windows 10 or 11. macOS / Linux are not supported in this fork — recorder, capture, launcher scripts, and proxy hooks are all Windows-specific.
  • Rust (stable MSVC toolchain) via rustup-init.exe. Pick default host triple x86_64-pc-windows-msvc, default toolchain stable, profile minimal. Make sure ~/.cargo/bin is on PATH.
  • Visual Studio Build Tools 2022 with the Desktop development with C++ workload and the Windows 11 SDK (22621 or newer): 
    winget install Microsoft.VisualStudio.2022.BuildTools `
  --override "--passive --add Microsoft.VisualStudio.Workload.VCTools `
              --add Microsoft.VisualStudio.Component.Windows11SDK.22621"
  • Node.js 20+ and npm.

Build

git clone https://github.com/Gansager/pluely-customized.git
cd pluely-customized
npm install
npm run tauri build

First clean build takes ~10 minutes (~500 Rust crates + vite frontend + bundler). Incremental Rust-only builds finish in ~2 minutes.

Outputs:

  • src-tauri/target/release/pluely.exe — the binary
  • src-tauri/target/release/bundle/msi/*.msi — Windows installer
  • src-tauri/target/release/bundle/nsis/*-setup.exe — NSIS installer

Install the freshly-built binary in place

If you already have stock Pluely installed and just want to swap the executable:

Stop-Process -Name pluely -Force -ErrorAction SilentlyContinue
Copy-Item "src-tauri\target\release\pluely.exe" "$env:LOCALAPPDATA\Pluely\pluely.exe" -Force
Start-Process "$env:LOCALAPPDATA\Pluely\pluely.exe"
The first time, install via the .msi / .exe so %LOCALAPPDATA%\Pluely\, registry entries, and the Start menu shortcut all get set up.

Optional: local Claude Code proxy

If you want to route Pluely through a local Claude Code CLI (no Anthropic API credits) and want screenshot support via Claude Code's Read tool, set up the companion proxy:

  1. Install @anthropic-ai/claude-code globally and authenticate it once interactively: 
    npm install -g @anthropic-ai/claude-code
claude  # one-time login
  2. Place the proxy scripts somewhere local (e.g. ~/pluely-proxy/). The proxy code is not in this repo — it's a private personal setup. The key file is proxy.py (HTTP server forwarding OpenAI-format requests to claude -p, with image-to-temp-file extraction).
  3. Configure a custom AI provider in Pluely Settings pointed at http://127.0.0.1:8765/v1/chat/completions.

Local STT (faster than cloud) uses Whisper:

pip install -U whisper-cpp-python  # or your preferred whisper.cpp binding
python whisper-server.py           # runs on http://127.0.0.1:8766

Then configure a custom STT provider in Pluely Settings pointed at the whisper-server.

Troubleshooting

  • System audio "Capture already running" error. The fork's auto-recovery should self-heal after ~250 ms. If it doesn't, restart Pluely.
  • Ctrl+Shift+A doesn't work. Some other app holds that hotkey on your machine. Change the Audio recording shortcut in Settings → Shortcuts.
  • Selection mode shows white screens. Already fixed in this fork — Selection mode uses the captured snapshot as the overlay background. If you still see this, you're on an older build.
  • Pluely doesn't respond to system audio. Check Windows has the right speaker / system audio device selected and audio is actually playing. WASAPI loopback returns silence when nothing is playing.
  • Reverting to stock Pluely. Copy any official pluely.exe v0.1.9 release over %LOCALAPPDATA%\Pluely\pluely.exe.
